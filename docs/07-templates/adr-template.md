# ADR Template

Copy into `docs/03-architecture/adr/NNNN-short-title.md`, numbered sequentially after the last existing ADR. ADRs are append-only — if a decision changes, write a new ADR that supersedes the old one (link both ways) rather than editing history.

```md
# ADR NNNN — <Decision title>

**Status:** Accepted | Superseded by ADR NNNN
**Date:** YYYY-MM-DD

## Context
What situation/problem prompted this decision?

## Decision
What was decided, stated plainly.

## Alternatives considered
What else was on the table, and why it wasn't chosen.

## Consequences
What this decision makes easier, harder, or constrains going forward.

## Follow-up actions
Concrete next steps this decision implies (e.g. "update docs/03-architecture/app-architecture.md §X").
```
