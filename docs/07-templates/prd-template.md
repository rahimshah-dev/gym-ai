# PRD Template

Copy this structure for a new feature-level PRD (for product-wide PRDs, extend `docs/01-product/prd/gymai-coach-prd.md` directly instead of creating a new file).

```md
# <Feature Name> PRD

**Status:** Draft | Approved | In progress | Shipped

## Problem
What issue exists now, for whom, and why now?

## Goal
The desired outcome if this succeeds.

## Non-goals
What's explicitly out of scope — as important as the goals.

## Users
Who this serves.

## User stories
User-centered scenarios ("As a ___, I want ___ so that ___").

## Requirements
Functional requirements (what it must do).

## Non-functional requirements
Performance, privacy, accessibility, reliability.

## Success metrics
How success will be measured.

## Dependencies
Systems, approvals, or teams required.

## Risks
Known concerns.

## Open questions
Unresolved topics.
```

See `docs/00-overview/repository-guide.md` "Field-by-field guidance" for what belongs in each field.
