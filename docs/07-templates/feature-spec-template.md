# Feature Spec Template

Lighter-weight than a full PRD — use for a well-understood feature that doesn't need the full problem-statement/goals treatment.

```md
# <Feature Name>

## Summary
One paragraph: what this is and why.

## Scope
What's included. What's explicitly not (link to the parent PRD's non-goals if relevant).

## Design
Link to the relevant docs/02-design/ pattern or component, or describe new UI needed.

## Implementation notes
Which packages/apps this touches, any architecture-boundary considerations (see root AGENTS.md).

## Acceptance criteria
Checklist of what "done" means.
```
