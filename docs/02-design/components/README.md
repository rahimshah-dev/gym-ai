# Component docs

One file per `packages/ui` component once it exists, documenting variants, states, and accessibility notes (see `design:design-system` skill's documentation pattern: naming, variants, states, a11y). Currently implemented components (`Button`, `Chip`, `Card` — see `packages/ui/README.md`) don't have dedicated docs yet since they're minimal ports; add a `<component-name>.md` here when a component's API grows complex enough to need documentation beyond its own source comments and Storybook stories.
